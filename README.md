# UserManagerQtNode

A simple client-server application demonstrating a Qt/QML GUI client interacting with a Node.js server backed by a SQLite database.  
The client supports adding, deleting, and displaying users, with live updates using WebSockets.

---

## **Technologies**

- **Client:** C++ / Qt 5.15.8, QML  
- **Server:** Node.js, Express.js, SQLite3  
- **Networking:** HTTP (GET, POST, DELETE) and WebSocket for live updates  
- **Data format:** JSON  

---

## **Features**

- Add new users via GUI  
- Delete existing users  
- Display users in a scrollable list  
- Live refresh when server updates or restarts using WebSockets

## **Setup Instructions**

### Server

1. Go to the server folder:

```bash
cd server
```

2. Install dependencies:
```
sh run_install.sh
```

3. Start the server
```
npm start
```

The server will run on http://localhost:3000 and WebSocket on port 3001

### Client

1. Open the client folder in Qt Creator or configure with CMake.

2. Build and run the project.

The client will automatically fetch existing users from the server and update the GUI in real time.

### Usage

- Enter Name and Age, click ADD to add a new user.

- Click REM to remove a user.

- The list updates automatically when the server sends WebSocket notifications.

### Notes

- WebSocket connection attempts are retried automatically if the server is not yet running.

- SQLite database file (users.db) is excluded from the repository using .gitignore.
